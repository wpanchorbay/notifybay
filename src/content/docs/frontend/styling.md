NotifyBay heavily uses Tailwind CSS for its modern UI components. However, using Tailwind inside WordPress requires some special configuration to avoid breaking the native dashboard styles.

## The Problem: Pre-flight Resets

Tailwind CSS includes a "pre-flight" stylesheet. This is a set of base styles designed to smooth over cross-browser inconsistencies. 

Unfortunately, if these pre-flight styles are loaded into the WordPress admin, they will aggressively reset the styling of native elements (like making all headings the same size, or removing borders from inputs), which breaks the look of the WordPress dashboard.

## The Solution: Pre-flight Guard

To solve this, NotifyBay uses a custom pre-flight guard. 

If you want an element (and all its children) to **ignore** the Tailwind reset and keep its native WordPress styling, you must add the `notifybay-ignore-preflight` class to it.

```jsx
// This div and everything inside it will KEEP native WP styles
<div className="notifybay-ignore-preflight">
  <h2>Native Heading</h2>
  <input type="text" className="regular-text" />
</div>
```

## The `notifybay-` Prefix

To prevent naming collisions with other plugins or themes that might also be using Tailwind, all utility classes generated by NotifyBay are prefixed with `notifybay-`.

Instead of writing `className="bg-blue-500 text-white"`, you must write:

```jsx
className="notifybay-bg-blue-500 notifybay-text-white"
```

> [!WARNING]  
> If you white-labeled the plugin using `rename.sh`, this prefix was likely changed. Make sure you are using your custom prefix! You can check your `tailwind.config.js` to confirm the current prefix.
